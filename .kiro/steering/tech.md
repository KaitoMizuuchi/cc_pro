# 技術スタック

## アーキテクチャ

フロントエンドとバックエンドを分離したSPA構成。フロントエンドはReact + Viteで構築し、バックエンドはHonoによるREST APIサーバーとする。データベースはDockerで起動するPostgreSQLを使用し、PrismaでORMアクセスする。

## コア技術

- **言語**: TypeScript（フロントエンド・バックエンド共通）
- **ランタイム**: Bun
- **フロントエンド**: React + Vite
- **バックエンド**: Hono
- **データベース**: PostgreSQL（Docker）
- **ORM**: Prisma

## 主要ライブラリ

開発パターンに影響を与える主要ライブラリのみ記載する。

| 役割 | ライブラリ | 用途 |
|------|-----------|------|
| ルーティング | react-router-dom | SPA内のページ遷移、認証ガード |
| サーバー状態管理 | TanStack Query | API通信のキャッシュ・再取得・ローディング状態管理 |
| フォーム管理 | react-hook-form | フォームの状態管理・バリデーション連携 |
| バリデーション | zod | スキーマ定義、フロント/バックエンド共通バリデーション |
| スタイリング | Tailwind CSS | ユーティリティファーストCSS |
| アイコン | lucide-react | UIアイコン |

## 開発標準

### 型安全性
- TypeScript strictモードを使用
- `any` 型の使用を避け、適切な型定義を行う
- zodスキーマからTypeScript型を導出するパターンを推奨

### コード品質
- **リンター/フォーマッター**: Biome（ESLint/Prettierの代替として統一的に使用）
- Biomeの設定に従い、一貫したコードスタイルを維持する

### テスト
- **テストフレームワーク**: Vitest
- ユニットテスト・統合テストをVitestで実行する

## 開発環境

### 必須ツール
- Bun（ランタイム・パッケージマネージャー）
- Docker / Docker Compose（PostgreSQL）

### 主要コマンド
```bash
# 開発サーバー起動: bun run dev
# ビルド: bun run build
# テスト実行: bun run test
# リント: bun run lint
# フォーマット: bun run format
# DB マイグレーション: bunx prisma migrate dev
```

## 主要技術判断

- **BunをNode.jsの代わりに採用**: 高速な起動・実行速度とビルトインのTypeScriptサポートを活用
- **Honoをバックエンドに採用**: 軽量・高速なWebフレームワークで、エッジランタイム互換性を持つ
- **BiomeをESLint+Prettierの代わりに採用**: リント・フォーマットを単一ツールで統合し、設定の簡素化と高速化を実現
- **zodによるスキーマファースト**: フロントエンド・バックエンド間でバリデーションロジックを共有する基盤として使用
- **TanStack Queryによるサーバー状態管理**: クライアント状態とサーバー状態を明確に分離し、キャッシュ・再取得戦略を統一
